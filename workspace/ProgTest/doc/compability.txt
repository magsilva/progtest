A ProgTest original utilizava, em sua porção Web, as bibliotecas: MyFaces
(versão 1.1), JSF Oracle ADF Faces, Apache Tomahawk, JSP Apache Tiles, htmLib,
JSTL (JavaServer Pages Template Library), uma tag library para transformações
XSLT desconhecida. A aplicação em si era executada com o servidor Apache
Tomcat, provavelmente em sua versão 4 ou 5.0.

Do período da construção inicial da ProgTest até os dias atuais, muito mudou
nas tecnologias Web disponíveis e diversas incompatibilidades da ProgTest com
ferramentas atuais aconteceram:

 Apache MyFaces: Implementação da especificação JSF pela Apache Foundation.
 Apesar da implementação da versão 1.1 da JSF ser razoável, a implementação da
 versão 1.2 é criticada, tendo os desenvolvedores preferência pela implementação
 de referência da Sun. A compabilidade do MyFaces com demais componentes (como
 o container Web) pode ser verificada em "http://myfaces.apache.org/compatibility.html".

 Oracle ADF Faces Components: O Oracle ADF Faces Components é uma parte do
 Application Development Framework (ADF) (este, por sua vez, um framework
 para o desenvolvimento de aplicações Web). No início de 2006, a Oracle doou
 uma parte do código do projeto para o projeto Trinidad da Apache Foundation.
 Ela, em paralelo, desenvolve o ADF Faces e faz contribuições ao Apache
 Trinidad.

 Apache Tomahawk: Componentes JSF compatíveis com a especificação JSF 1.1 (ou
 seja, podem ser utilizados com qualquer implementação da JSF compatível com a
 versão 1.1, seja a implementação de referência da SUN ou o MyFaces do projeto
 Apache). 
 
 Apache Tiles: o Tiles é um mecanismo de gabaritos (templates) para aplicações
 Web. A versão utilizada na ProgTest é a 1.1 e atualmente a versão em desenvolvimento
 é a 2.0. Na verdade, na versão 1.1, o Tiles era o Struts Tiles. Apenas com a
 versão 2.0 é que separou-se o projeto Tiles do Struts.
 Ps: Ainda existe o Shale-Tiles (Shale é a continuação do projeto Struts).
 
 HTML Layout Tag Library: biblioteca de marcadores HTML (jstl tag library).


Recomendações
-------------

No espírito de uma solução de software livre, a substituição do Oracle ADF Faces
por uma solução é mais do que desejável, é um requisito. Por sorte do destino,
a Oracle adiantou-se quanto a isso, com a doação de código para o projeto Trinidad.
Assim, deve-se proceder com a substituição do Oracle ADF Faces pelo Apache Trinidad
o quanto antes.

Atualizar do Struts Tiles 1.1 para o Tiles 2.0 também é desejável. O Struts Tiles
não é mais desenvolvido.

Quanto à implementação da especificação da JSF 1.2, existe um consenso
quanto a superior qualidade da implementação de referência da Sun. Como ela também
é software livre (licença CDDL, similar à MPL), sua adoção é perfeitamente possível.

A "tag library para transformações XSLT desconhecida" pode ser perfeitamente
substituída pela biblioteca padrão da JSTL.

O htmLib não pode ser substituído e, na verdade, nem é necessária a sua substituição.

Quanto ao Tomcat, qualquer implementação da JSF 1.2 requer um container Web que
implemente as especificações JSP 2.1 e Servlet 2.5. O Tomcat 6.0 implementa essas
especificações. 


 



Compatibilidade do Apache Tomahawk
----------------------------------

Como dito anteriormente, o Tomahawk foi desenvolvido com a versão 1.1 da
especificação JSP em mente. A tabela que relaciona a versão dos programas e a
compatibilidade (http://wiki.apache.org/myfaces/CompatibilityMatrix) mostra a
situação atual. Aparentemente implementações da JSP 1.2 não são suportadas.

Observando o código-fonte do projeto, ele é bem distinto dos demais projetos
desenvolvidos pelo Apache Foundation. Ele não utiliza o Maven, não possui uma
documentação organizada e atualizada. Esses dois últimos itens são um forte
indicativo de imaturidade, dado que é difícil para membros da comunidade
compilar novas versões sem a documentação e a utilização do Maven.

Pessoalmente, eu recomendaria a não utilização do Tomahawk. No entanto, se
funcionar do jeito que está, melhor.


Compatibilidade do Apache Trinidad
----------------------------------

Como dito anteriormente, o Tomahawk foi desenvolvido com a versão 1.1 da
especificação JSP em mente. O MyFaces 1.1 e 1.2, que implementam, respectivamente,
a versão 1.1 e 1.2 do JSF são suportados, bem como o SUN RI. Ironicamente,
o Apache Trinidad foi mais testado com o SUN RI do que com o MyFaces.
 

Compabilidade do Apache Tiles
-----------------------------

Instruções para a migração do Struts Tiles (1.1) para o Tiles 2.0 encontram-se
em "http://tiles.apache.org/migration/".
